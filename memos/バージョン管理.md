## バージョン体系（v4.7.4以降）

```
v{Phase}.{Feature}.{Fix}-{サフィックス}
```

| 桁 | 意味 | カウント方法 |
|---|------|------------|
| 左（Phase） | 現在のPhase番号 | 開発ロードマップのPhase番号 |
| 中（Feature） | Phase内で追加した機能の数 | 機能追加ごとにカウントアップ |
| 右（Fix） | 修正した回数 | バグ修正ごとにカウントアップ |
| サフィックス | stable / pre-phaseX 等 | 安定版 / ロールバックポイント |

例:
- `v4.10.3-stable` → Phase 4、10機能追加、3回修正、安定版
- Phase 5に進んだら → `v5.0.0` からスタート

**注意:** v4.0.0以前は旧体系（メジャー.マイナー.パッチ）で採番されている。

**運用ルール:** バージョンアップ時は必ず以下を同時に実行すること:
1. `config/app_config.py` の `APP.version` を更新
2. このファイルのバージョン一覧・詳細を更新
3. Gitタグを付与してプッシュ
