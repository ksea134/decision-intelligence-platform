## バージョン体系（v4.7.4以降）

```
v{Phase}.{Feature}.{Fix}-{サフィックス}
```

| 桁 | 意味 | カウント方法 |
|---|------|------------|
| 左（Phase） | 現在のPhase番号 | 開発ロードマップのPhase番号 |
| 中（Feature） | Phase内で追加した機能の数 | 機能追加ごとにカウントアップ |
| 右（Fix） | 修正した回数 | バグ修正ごとにカウントアップ |
| サフィックス | stable / pre-phaseX 等 | 安定版 / ロールバックポイント |

例:
- `v4.7.4-stable` → Phase 4、7機能追加、4回修正、安定版
- Phase 5に進んだら → `v5.0.0` からスタート

**注意:** v4.0.0以前は旧体系（メジャー.マイナー.パッチ）で採番されている。

## 各バージョンの詳細

### v4.7.4-stable（現行）

最新の安定版。

**機能追加（7件）:**
1. ADKエージェント基盤（Phase 4A）
2. Gemini Flash/Pro使い分け（Phase 4B）
3. キャッシュ最適化 + ADKフォールバック
4. GCS資料のADKエンジン連携
5. スマートカードCSV管理方式
6. スマートカードデータソース指定
7. UIラベル追加（DI Dashboard/Conversation/ナビゲーション/プロンプト/処理概要/データ取得先）

**バグ修正（4件）:**
- #004: logger未定義NameError
- #005: ADKエンジンでGCS資料が反映されない
- #006: 企業間の過去事例混入（汚染データ削除）
- #007: ADKエージェントinstruction累積（空応答・企業間混入）

**ドキュメント整備:**
- 7層別仕様書（9ファイル）
- 開発ルール（実装計画書・テスト計画書・結果報告書の必須化）
- セキュリティ＆ガバナンス要件
- AI Ops＆オブザーバビリティ要件
- バージョン体系の統一（v{Phase}.{Feature}.{Fix}）

### v4.0.0-stable

Phase 4A完了。ADKエージェント基盤を実装。USE_ADK_ENGINEフラグで従来エンジンとの切替可能。

### v3.6.1-stable

Phase 3完了。Vertex AI Search統合（Q&A長期記憶、セマンティック検索、企業単位フィルタ）。ファビコン追加、タイトルデザイン変更。

### v2.2.0-stable

v4完全再設計。

**アーキテクチャ変更:**
- function callingを完全に廃止（mode=ANY/AUTOの不安定性を根本解決）
- BQデータ取得をSELECT * FROM table固定（SQL生成の不確実性を排除）
- Router分類をキーワードのみに変更（LLM分類のAPI遅延・障害を排除）
- stream_eventsをゼロから再設計（whileループ廃止、2フェーズ固定構造）

**バグ修正:**
- #001: 日本語カラム名の自動バッククォート付与
- #002: Geminiがツールを呼ばずテキスト回答を返す問題
- #003: mode=ANY+ストリーミングによる応答遅延・SQL無限繰り返し

### v2.1.0-demo-ready

13:30のお客様デモ用に確保した版。デモは成功。

### v2.0.0-phase2-baseline

Phase 1完了後、Phase 2（Memory Store）開始前に確保した安定地点。Phase 2はスキップが決定。

### v1.1.0-phase1

Phase 1（Agent Router）完了時点。LangGraphでの意図分類基盤を構築。
