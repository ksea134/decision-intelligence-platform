# エージェントAI（Agent Router）導入のメリット

> 作成日: 2026-03-18
> 文脈: DIP Phase 1 で構築した Agent Router の導入理由を整理

---

## 現状（単一エージェント）

1つのGeminiが全質問に対して同じプロンプト・同じ処理で回答。

## Agent Router導入のメリット

### 1. 回答精度の向上
- 専門エージェントが専門の指示を持つ。「売上が落ちた原因は？」に対して、要因分析エージェントは「比較すべき期間を特定し、変動要因を構造的に分解せよ」という専門指示で動く
- ハルシネーション（でっち上げ）の抑制。タスク固有のガードレールを設定できる

### 2. 回答の質の分化
- 「売上教えて」（汎用）と「なぜ売上が下がったのか」（分析）で最適な回答形式に分化
- 分析は原因→影響→提言の構造、比較は表形式、など

### 3. 将来のモデル使い分け（Phase 4B）
- 汎用質問 → Gemini Flash（速い・安い）
- 複雑な分析 → Gemini Pro（精度重視）
- 高度な推論 → Claude Opus（最高精度）
- 質問の種類に応じてコストと精度を最適化

### 4. 拡張性
- 新しいエージェント（レポート生成、異常検知等）を既存に影響なく追加可能
- 単一エージェントだとプロンプト肥大化で性能劣化する

### 5. デバッグ・改善の容易さ
- 「比較の回答がおかしい」→ 比較エージェントだけ修正すればよい
- 1箇所の修正が全質問に影響しない

## 回答速度について
- 速くはならない（Router分類のAPI呼び出しが1〜2秒追加）
- 将来的にモデル使い分けで汎用質問にFlashを使えば、その分は速くなる

## まとめ

| 項目 | 効果 |
|------|------|
| 回答精度 | 上がる（専門指示による的確な分析） |
| 回答速度 | 変わらない〜やや遅い（分類の追加コスト） |
| 回答品質 | 上がる（タスク別に最適な回答形式） |
| コスト最適化 | 将来的に可能（モデル使い分け） |
| 拡張性 | 高い（エージェント追加が容易） |
| 保守性 | 高い（問題の切り分けが容易） |
