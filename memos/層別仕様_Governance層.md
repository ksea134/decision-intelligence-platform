# DIP Governance層 仕様

> 作成日: 2026-03-19
> ステータス: 未着手（Phase 6）
> 関連: `memos/層別仕様_Security＆Governance層.md`（セキュリティ対策の全体像）

---

## 1. 役割

データカタログ、アクセス制御、データリネージュを実現し、企業データのガバナンスを担保する。セキュリティ＆ガバナンス要件で定義した「第2段階: Dataplex」に該当。

## 2. 技術要素

| サービス | 役割 |
|---------|------|
| **Dataplex** | データカタログ、ゾーン管理、ポリシー一元管理、リネージュ |
| **BigQuery Row-level Security** | 行レベルのアクセス制御 |
| **BigQuery Column-level Security** | 列レベルのアクセス制御 |
| **Cloud DLP** | 機密データの自動検出・マスキング |

## 3. 機能詳細

| # | 機能 | 説明 | 技術要素 |
|---|------|------|----------|
| G01 | データカタログ | BQ/GCSリソースの自動登録・検索 | Dataplex Catalog |
| G02 | ゾーン管理 | 企業別データゾーンの定義と分離 | Dataplex Zones |
| G03 | 行レベルアクセス制御 | ロールに応じて参照可能な行を制限 | BigQuery Row-level Security |
| G04 | 列レベルアクセス制御 | 機密列（利益率等）をロールで制限 | BigQuery Column-level Security + Authorized Views |
| G05 | 機密データマスキング | 個人情報・機密情報の自動検出と無害化 | Cloud DLP |
| G06 | データリネージュ | データの出自・変換履歴の追跡 | Dataplex Lineage |
| G07 | 監査ログ | 誰がいつ何にアクセスしたか | Cloud Audit Logs |

## 4. 3段階アクセス制御モデルにおける位置

```
第1段階: エージェント → どのデータベースにアクセスできるか
第2段階: Dataplex（本層）→ どのデータを受け渡してよいか  ← ここ
第3段階: Vertex AI Search → 過去Q&Aをどの範囲で共有するか
```

→ 3段階モデルの全体像は `memos/層別仕様_Security＆Governance層.md` セクション5

## 5. 機密度分類

| 機密度 | データ例 | アクセス制御 |
|--------|---------|-------------|
| 一般 | 商品マスタ、公開済み売上 | 全ロールアクセス可 |
| 社外秘 | 部門別利益率、顧客リスト | 特定ロール以上 |
| 極秘 | 内部通報、監査指摘、M&A情報 | 経営層のみ。Four Eyes原則 |
| 法規制対象 | 個人情報、財務報告データ | 法定要件に準拠 |
