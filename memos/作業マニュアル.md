## 1. DIPの起動（ローカル開発環境）

ターミナルを**2つ**開いて、それぞれ実行する。

### 1-1. バックエンド（FastAPI）

```bash
cd ~/dip && source .venv/bin/activate && uvicorn backend.main:app --reload --port 8000
```

成功すると以下が表示される：
```
INFO:     Uvicorn running on http://127.0.0.1:8000
INFO:     Application startup complete.
```

**このターミナルは閉じないこと。** 閉じるとバックエンドが止まる。

### 1-2. フロントエンド（Next.js）

新しいターミナルタブ（**Cmd + T**）で：

```bash
cd ~/dip/frontend && npm run dev 2>/dev/null
```

> `2>/dev/null` は**必須**。つけないとエラーログ大量出力でMacがフリーズする可能性がある（バグ#012）

成功すると以下が表示される：
```
▲ Next.js (Turbopack)
- Local: http://localhost:3000
✓ Ready
```

### 1-3. ブラウザで確認

```
http://localhost:3000
```

「意思決定支援AI」の画面が表示され、左側のメニューから企業を選べる。

### 1-4. 終了するとき

1. フロントエンドのターミナルタブで **Ctrl + C**
2. バックエンドのターミナルタブで **Ctrl + C**
3. **Cmd + Q** でターミナル全体を終了

## 3. トラブルシューティング

### 「ポートが使用中」と出たら

```bash
lsof -i :8000   # バックエンドのポート確認
lsof -i :3000   # フロントエンドのポート確認
kill [PID]       # 表示されたPIDを停止
```

### 企業が選べない・データが出ない

- バックエンドが起動しているか確認（ターミナルに`Application startup complete.`が出ているか）
- ポートが正しいか確認（バックエンド: **8000**、フロントエンド: **3000**）
- `--port 8080` ではなく `--port 8000` で起動すること

### 回答が遅い（30秒以上）

- データカタログのテーブル選択が動作しているか確認（ログに`[DataCatalog] AI selected`が出ているか）
- BigQueryテーブルに説明を追加すると、テーブル選択精度が上がり速度が改善する（セクション7参照）

### インフォグラフィックが表示されない

- 回答生成後、数秒待つと補足フェーズが実行される
- 「補足情報を整理しています…」のスピナーが表示されるまで待つ
- Geminiの出力が確率的にばらつくため、再度質問すると表示される場合がある

### Macがフリーズした

1. 電源ボタンを **10秒長押し** して強制シャットダウン
2. 再起動後、セクション1の手順で起動し直す
3. フロントエンド起動時は必ず `2>/dev/null` をつける

### Cloud Runで「API key」エラーが出る

- Cloud RunはVertex AI経由（ADC認証）でGeminiを呼び出す。APIキーは不要
- エラーが出た場合はCloud Runのサービスアカウントに `Vertex AI User` ロールが付与されているか確認

## 6. 新しい企業を追加する

### Step 1: ローカルファイルの準備

#### 1-1. companies.csv に登録

`dip/data/companies.csv` に1行追加する。

```csv
企業名,フォルダ名
新しい企業名,folder_name
```

- **企業名**: サイドバーに表示される名前（日本語OK）
- **フォルダ名**: 半角英数字+アンダースコア。BigQueryのデータセットIDと一致させること

#### 1-2. 企業フォルダを作成

`dip/data/{フォルダ名}/` 配下に以下のフォルダ構成を作成する。

```
dip/data/{フォルダ名}/
├── introduction/        ← 「はじめに」に表示されるテキスト
│   └── readme.md
├── knowledge/           ← 企業の前提知識（業界情報、用語集など）
│   └── *.md / *.txt
├── prompts/             ← AIの回答スタイル・役割の定義
│   └── prompt.md        ← 必須
├── smart_cards/         ← スマートカード（ショートカットプロンプト）
│   ├── smart_cards.csv  ← カード一覧定義
│   └── *.md             ← 各カードのプロンプト
├── structured/          ← CSVなどの構造化データ
│   └── *.csv
└── unstructured/        ← テキスト・報告書などの非構造化データ
    └── *.txt / *.md
```

**最低限必要なファイル:** `prompts/prompt.md`

### Step 2: BigQueryの設定

1. https://console.cloud.google.com/bigquery を開く
2. プロジェクト `decision-support-ai` を選択
3. 「データセットを作成」→ データセットID = companies.csvのフォルダ名
4. リージョン: `asia-northeast1`（東京）
5. テーブルを作成してデータをアップロード
6. **テーブルに説明を追加する**（セクション7参照 — AIのテーブル選択精度が上がる）

**注意:**
- カラム名に日本語使用OK（DIPが自動でバッククォートを付与）
- データセットIDとcompanies.csvのフォルダ名が一致していないとデータが取得できない

### Step 3: GCS（Cloud Storage）の設定

1. https://console.cloud.google.com/storage を開く
2. バケット `dsa-knowledge-base` を選択
3. `{フォルダ名}/unstructured/` フォルダを作成
4. テキストファイル（.txt / .md）をアップロード

**注意:**
- フォルダ名はcompanies.csvと一致させること
- ファイルサイズ: 5MB以内
- 対応拡張子: .txt, .md

### Step 4: 動作確認

1. DIPを起動（ローカルまたはCloud Runにデプロイ）
2. サイドバーで新しい企業を選択
3. 右カラムの「データ取得先」→「データソース稼働状況」を確認
   - BigQuery: 接続済み（テーブル数が表示される）
   - GCS: 接続済み（ファイル数が表示される）
4. 質問を送信して回答が返ることを確認

## 7. BigQueryテーブルに説明を追加する

### なぜやるの？

DIPのAIが「この質問にはどのテーブルを使うべきか」を判断するとき、Dataplex Data Catalogのテーブルの**説明**を見て判断する。説明がないと、カラム名（date, area, severity等）だけで判断するので精度が落ちる。

### 手順

**ステップ1**: https://console.cloud.google.com/bigquery を開く

**ステップ2**: 左側のエクスプローラーで `decision-support-ai` → データセット（例: `demo_factory02`）→ テーブル（例: `incident_records`）をクリック

**ステップ3**: 上部の「詳細」タブをクリック

**ステップ4**: 「説明」の横にある鉛筆アイコン（編集）をクリック

**ステップ5**: テーブルの説明を日本語で入力して「保存」

例：
```
インシデント・事故記録。安全パトロールで発見された指摘事項、重大度、負傷の有無を記録。
```

**ステップ6**: 他のテーブルも同様に繰り返す

### 説明の書き方のコツ

| 良い例 | 悪い例 |
|--------|--------|
| 生産実績。日別・ライン別の計画数量と実績数量 | production results |
| 設備アラート。異常検知時の重大度とメッセージ | alerts |
| 従業員の資格取得状況と有効期限 | cert |

**ポイント:**
- 日本語で書く
- 「何のデータか」「何が入っているか」を1〜2文で
- DIPのAIがこの説明を読んでテーブルを選ぶので、具体的に書くほど精度が上がる

### 確認方法

説明を追加した後、DIPで質問すると、バックエンドのログに以下が表示される：

```
[DataCatalog] AI selected 5/12 tables: [...]
```

### 補足

- 説明を追加しなくてもDIPは動く（カラム名で判断）
- 説明を追加すると**テーブル選択の精度が上がり、応答速度も改善**する
- テーブルを新しく作成した場合、DIPが**自動的に検出**する（Dataplex Data Catalog APIがBigQueryテーブルを自動認識）
- JSONファイルの手動更新は不要

### テーブル説明の有無による精度の違い

| 状態 | テーブル選択精度 | 備考 |
|------|----------------|------|
| 説明あり（日本語） | **高い** | 推奨 |
| 説明なし + カラム名が意味のある英語 | **中程度** | `incident_records`, `production_results` 等 |
| 説明なし + カラム名が略語・社内用語 | **低い** | `t_mst_001`, `dept_cd`, `flg` 等。全テーブル取得にフォールバックする可能性あり |

**お客様の本番データは略語や社内用語のカラム名であることが多い。テーブルの説明を追加することを強く推奨する。これはお客様への提案ポイントにもなる。**

## 8. フィードバックの確認方法

DIPの回答に対するユーザーフィードバック（👍👎）は以下で確認できる。

### Cloud Logging（リアルタイム）

1. https://console.cloud.google.com/logs を開く
2. クエリ欄に `[Feedback]` と入力して「クエリを実行」

### BigQuery（SQL分析）

```sql
SELECT timestamp, rating, company, comment, question
FROM `decision-support-ai.dip_ops.v_feedback`
ORDER BY timestamp DESC
LIMIT 100
```

### Looker Studio（ダッシュボード）

https://lookerstudio.google.com でフィードバック分析ダッシュボードを開く。

- V1エンジン（スマートカード）と補足フェーズ: Claude利用可能
- ADKエンジン（チャット入力）: Gemini専用（Vertex AI Model GardenでのClaude有効化で対応予定）

## 10. エージェント設定の変更

エージェントの定義は `config/agents.json` に外部化されている。コードを変更せずにエージェントの追加・プロンプト変更が可能。

### 変更手順

1. `config/agents.json` を編集
2. バックエンドを再起動（ファイル変更時に自動リロード）

### 設定項目

```json
{
  "name": "analysis_agent",
  "display_name": "要因分析エージェント",
  "model_role": "deep",
  "triggers": ["なぜ", "原因", "理由", "要因"],
  "instruction": "あなたは要因分析の専門家です。..."
}
```

- **display_name**: サイドバーとAI処理フローに表示される名前
- **model_role**: 使用するモデル（deep / fast / router / supplement）
- **triggers**: このエージェントが選ばれるキーワード
- **instruction**: AIへの指示文（プロンプト）
