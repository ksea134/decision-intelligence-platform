## 1. 新しい企業を追加する

### Step 1: ローカルファイルの準備

#### 1-1. companies.csv に登録

`dip/data/companies.csv` に1行追加する。

```csv
企業名,フォルダ名
新しい企業名,folder_name
```

- **企業名**: サイドバーに表示される名前（日本語OK、記号OK）
- **フォルダ名**: 半角英数字+アンダースコア。BigQueryのデータセットIDと一致させること

#### 1-2. 企業フォルダを作成

`dip/data/{フォルダ名}/` 配下に以下のフォルダ構成を作成する。

```
dip/data/{フォルダ名}/
├── introduction/        ← 「はじめに」に表示されるテキスト
│   └── readme.md        ← AIの取り扱い注意事項など
├── knowledge/           ← 企業の前提知識（業界情報、用語集など）
│   └── *.md / *.txt
├── prompts/             ← AIの回答スタイル・役割の定義
│   └── prompt.md        ← 必須。回答フォーマット・見出しルールなど
├── smart_cards/         ← チャット画面のショートカットプロンプト
│   ├── alert.md         ← 緊急アラート
│   ├── hint.md          ← ヒント
│   ├── kpi.md           ← KPI確認
│   ├── notable.md       ← 気になる兆候
│   └── recent.md        ← 最新動向
├── structured/          ← CSVなどの構造化データ
│   └── *.csv
├── templates/           ← 質問テンプレート
│   └── questions.txt
└── unstructured/        ← テキスト・報告書などの非構造化データ
    └── *.txt / *.md
```

**最低限必要なファイル:**
- `prompts/prompt.md` — これがないとAIの回答スタイルが未定義になる

### Step 2: Google Cloud（BigQuery）の設定

#### 2-1. BigQueryにデータセットを作成

1. Google Cloud Console にログイン
2. BigQuery を開く
3. プロジェクト `decision-support-ai` を選択
4. 「データセットを作成」をクリック
5. **データセットID**: companies.csv の「フォルダ名」と同じ名前にする
6. **リージョン**: `asia-northeast1`（東京）を選択
7. 作成をクリック

#### 2-2. テーブルを作成・データをアップロード

1. 作成したデータセットを選択
2. 「テーブルを作成」をクリック
3. CSVファイルをアップロード、またはスキーマを定義してデータを投入
4. テーブル名は任意だが、内容がわかる名前を推奨

**注意事項:**
- カラム名に日本語を使用可能（DIPが自動でバッククォートを付与する）
- SELECT文のみ実行される（INSERT/UPDATE/DELETE等は禁止されている）
- データセットIDとcompanies.csvのフォルダ名が一致していないとデータが取得できない

### Step 3: Google Cloud（GCS）の設定

#### 3-1. GCSにフォルダを作成

1. Google Cloud Console にログイン
2. Cloud Storage を開く
3. バケット `dsa-knowledge-base` を選択
4. `{フォルダ名}/unstructured/` フォルダを作成
5. テキストファイル（.txt / .md）をアップロード

**注意事項:**
- GCSのフォルダ名もcompanies.csvのフォルダ名と一致させること
- ファイルサイズ制限: 5MB以内
- 対応拡張子: .txt, .md

### Step 4: 動作確認

1. DIPを起動: `streamlit run app.py`
2. サイドバーで新しい企業を選択
3. 右側パネルで「データソース稼働状況」を確認
   - BigQuery: 接続済み（テーブル数が表示される）
   - GCS: 接続済み（ファイル数が表示される）
4. 「売上教えて」などの質問を送信して回答を確認

## 2. DIPの起動・停止

### 起動

```bash
# 既存プロセスを停止してから起動（必須）
pkill -f streamlit
streamlit run app.py
```

### 停止

```bash
pkill -f streamlit
```

**注意**: 複数のStreamlitプロセスが同時に動くと応答が極端に遅くなる。必ず`pkill -f streamlit`で停止してから起動すること。

## 3. トラブルシューティング

### 「データ取得できない」と表示される

- BigQueryのデータセットIDとcompanies.csvのフォルダ名が一致しているか確認
- Google Cloud Console でデータセットにテーブルが存在するか確認
- サイドバーの「GCP Project ID」が `decision-support-ai` になっているか確認

### 回答が遅い（30秒以上）

- `pkill -f streamlit` で全プロセスを停止してから再起動
- Gemini APIの一時的な負荷の可能性あり。数分待って再試行

### インフォグラフィックが表示されない

- 回答生成後、数秒待つと補足フェーズが実行される
- 「補足情報を整理しています…」のスピナーが表示されるまで待つ
