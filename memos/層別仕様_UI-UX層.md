## 3. スマートカード仕様

### CSV管理方式（2026-03-19刷新）

スマートカードは企業ごとのCSVファイルで管理。ハードコード定義（`_smart_card_defaults.py`）は廃止済み。

**CSV形式:** `data/{企業名}/smart_cards/smart_cards.csv`
```csv
企業コード,コード,アイコン,タイトル,データソース
mazda,recent,⚡,週間ニュース,all
mazda,daily_report,📋,営業日報まとめ,gcs:営業日報
mazda,kpi,📊,KPI確認,bq
```

| 要素 | カスタマイズ | 管理場所 |
|------|-----------|---------|
| カード数 | ✅ 企業ごとに0〜25枚 | CSVの行数で決まる |
| アイコン・タイトル | ✅ 企業ごとにカスタマイズ可 | CSV |
| プロンプト（質問文） | ✅ 企業ごとにカスタマイズ可 | `data/{企業名}/smart_cards/{コード}.md` |
| データソース | ✅ カードごとに指定可 | CSV（データソース列） |

**データソース列の値:**

| 値 | Geminiに渡されるデータ |
|---|----------------------|
| `all`（またはに空欄） | 全データソース（BQ + GCS + ローカル） |
| `bq` | BigQueryのみ |
| `gcs` | GCSのみ |
| `bq+gcs` | BQとGCS |
| `gcs:キーワード` | ファイル名にキーワードを含むGCSファイルのみ |
| `bq:キーワード` | テーブル名にキーワードを含むBQテーブルのみ |

**設計思想:** Geminiに「このデータだけ使え」と指示するのではなく、そもそも不要なデータを渡さない。LLMの不確実性はコード側で吸収する（開発ルールC03）。

**ルール:**
- CSVなし → 0枚（表示なし）
- 最大25枚（26個目以降は切り捨て）
- 1行5個のグリッド表示（最大5行）

### 表示タイミング
- チャット履歴が空のとき（会話開始時）だけ表示
- 一度質問すると消える
- スマートカード経由の質問は質問履歴に追加されない

### データフロー

```
1. アプリ起動 → 企業選択
   ↓
2. data/{企業名}/smart_cards/ からプロンプト読込
   - {id}.md があれば → そのファイルの内容をプロンプトに使用
   - なければ → _smart_card_defaults.py のデフォルトプロンプトを使用
   - ファイルが空なら → 安全策としてデフォルトを使用（空プロンプトでAPIエラー防止）
   ↓
3. チャット画面に5つのボタンを表示（5列レイアウト）
   ↓
4. ユーザーがボタンをクリック
   - st.session_state["pending_prompt"] にプロンプトをセット
   - st.session_state["pending_display"] に表示ラベル（アイコン+タイトル）をセット
   - st.rerun() で画面を再描画
   ↓
5. 再描画時にpending_promptを検出 → AIにプロンプトを送信
   - is_smart_card=True フラグ付きで実行
   - 質問履歴には追加されない
   ↓
6. AIが回答を生成（通常の質問と同じフロー）
```

### 管理ファイル

| ファイル | 役割 |
|---------|------|
| `infra/_smart_card_defaults.py` | 5つのカードのID、アイコン、タイトル、説明、デフォルトプロンプト |
| `infra/file_loader.py` | `load_smart_cards()` — 企業フォルダからプロンプト上書き読込 |
| `data/{企業名}/smart_cards/{id}.md` | 企業別のカスタムプロンプト |
| `ui/components/chat.py` | `_render_smart_cards()` — ボタン描画とクリック処理 |
| `domain/models.py` | `CompanyAssets.smart_cards` — データ構造定義 |
