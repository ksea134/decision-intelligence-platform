# DIP チャット対話デモシナリオ（3ステップ版）
## 工場長の朝 〜 3分で意思決定へ

---

## シナリオ概要

**設定**: 2026年3月20日（木）朝8:30、工場長の田中さんが出勤

**所要時間**: 約5分

**コンセプト**: DIPの強みは「1回の質問で網羅的な分析」。小出しにせず、複合的な質問で一気に核心へ。

---

## 対話シナリオ（3ステップ）

### Step 1: 状況把握 + 原因特定（1分）

**入力**:
```
おはよう。今日の工場の状況を教えて。問題があれば原因まで分析して。
```

**DIPが1回で返す内容（期待）**:
- 全ライン（A/B/C/D）の生産達成率
- **Cラインの異常検出**（達成率50%）
- 原因として**WLD-003（溶接ロボット）の故障**を特定
- 設備アラートの状況
- 過去の経緯（いつから悪化したか）

**デモでの強調ポイント**:
> 「1回の質問で、問題の発見から原因特定まで完了しています。BigQueryの生産データ、設備アラート、点検報告書を横断的に分析しています。」

---

### Step 2: 影響の全体像（1分）

**入力**:
```
WLD-003の問題が、品質・安全・顧客にどう影響しているか分析して。
```

**DIPが1回で返す内容（期待）**:
- **品質影響**: 溶接不良率の推移、工程別不良マップ
- **安全影響**: ヒヤリハット件数、具体的な事象（火花飛散、異常動作）
- **顧客影響**: TOYOTAからのクレーム履歴、納期リスクのあるオーダー
- **コスト影響**: 手直し・廃棄コスト、ダウンタイムコスト試算
- **リスク評価**: 各影響の深刻度

**デモでの強調ポイント**:
> 「構造化データ（BigQuery）と非構造化データ（GCS上の報告書）を組み合わせて、影響を多角的に分析しています。特にヒヤリハット報告書はテキストから自動抽出しています。」

---

### Step 3: 意思決定 + アクション（1分）

**入力**:
```
根本原因を整理して、今日私がすべきことを優先順位付きで教えて。
```

**DIPが1回で返す内容（期待）**:

**根本原因の整理**:
- WLD-003の経年劣化（振動系統の異常）
- 3/12の「要経過観察」判断が分岐点だった
- 予知保全体制の不備が拡大を招いた

**アクションプラン（優先順位付き）**:

| 優先度 | 時間軸 | アクション |
|--------|--------|-----------|
| 🔴 最優先 | 本日AM | WLD-003の即時停止判断 |
| 🔴 最優先 | 本日AM | A/B/Dラインへの振り分け検討 |
| 🟠 高 | 本日中 | TOYOTAへの状況説明・納期調整 |
| 🟠 高 | 今週中 | 該当ロットの全数検査 |
| 🟡 中 | 今月中 | 予知保全体制の見直し |

**デモでの強調ポイント**:
> 「分析だけでなく、具体的なアクションプランまで提案しています。これが『意思決定支援AI』の本質です。」

## デモ実施ガイド

### タイムライン

| 時間 | 内容 |
|------|------|
| 0:00-0:30 | DIP画面紹介、企業選択 |
| 0:30-1:30 | Step 1 実行・解説 |
| 1:30-2:30 | Step 2 実行・解説 |
| 2:30-3:30 | Step 3 実行・解説 |
| 3:30-5:00 | Q&A、追加質問デモ |

### 強調すべきDIPの価値

1. **網羅性**: 1回の質問で複数データソースを横断分析
2. **速度**: 従来なら数時間かかる分析を数十秒で
3. **統合**: 構造化データ（数値）+ 非構造化データ（報告書）
4. **意思決定**: 分析で終わらず、アクションまで提案

### トラブル時の対応

| 状況 | 対応 |
|------|------|
| WLD-003が出てこない | 「Cラインの設備を詳しく」と追加質問 |
| 非構造化データが引用されない | 「点検報告書の内容も含めて」と明示 |
| アクションが抽象的 | 「より具体的に、担当者レベルで」と追加 |
