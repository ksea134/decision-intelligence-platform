## 3. 問題の核心

### 3-1. 発生している事象

> **Cラインの溶接ロボット WLD-003 に劣化兆候が発生**
>
> 3月10日頃から徐々に症状が悪化し、生産性・品質・安全のすべてに影響が波及している。

### 3-2. 影響の連鎖
```
WLD-003の劣化
    │
    ├─→ 生産性低下（Cライン達成率 95%→50%）
    │
    ├─→ 品質悪化（溶接不良の増加、顧客クレーム）
    │
    ├─→ 設備アラート増加（WARNING/CRITICAL多発）
    │
    ├─→ エネルギー消費異常（+15〜20%）
    │
    └─→ 安全リスク（異常動作、火花飛散のヒヤリハット）
```

### 3-3. タイムライン

| 日付 | 出来事 |
|------|--------|
| 3/1 | 月次安全パトロール（異常なし） |
| 3/10 | **WLD-003の劣化兆候が顕在化開始** |
| 3/12 | 点検報告：振動2.3mm/s、軽微な異音 |
| 3/12 | ヒヤリハット：火花飛散 |
| 3/14 | 点検報告：振動2.8mm/s、異音増加 |
| 3/14 | ヒヤリハット：ロボット異常動作 |
| 3/15 | 安全パトロール：WLD-003安全柵の緩み指摘 |
| 3/16 | 点検報告：振動3.2mm/s、**基準値超過** |
| 3/16 | ヒヤリハット：異音発生 |
| 3/18 | 点検報告：振動3.5mm/s、**緊急対応必要** |
| 3/19 | **本日：デモ当日** |

## 4. デモシナリオ（実演手順）

### シーン1: 朝の状況確認（生産管理視点）

**ストーリー**: 工場長の田中さんが出勤後、まず昨日の生産状況を確認する。

**使用するスマートカード**: `昨日の生産実績サマリ`

**操作手順**:
1. 「製造業デモ02｜工場・生産現場」を選択
2. 「昨日の生産実績サマリ」をクリック

**期待される回答のポイント**:
- 全体の計画達成率が目標95%を下回っている
- **Cラインの達成率が約50%と極めて低い**
- 溶接ロボットWLD-003の故障が主因と特定
- 推奨アクション：WLD-003の即時停止と本格修理

**確認できるデータ**:
- BigQuery: `production_results` テーブル
- 参照ファイル数: 約10件

---

### シーン2: 設備状態の深掘り（保全視点）

**ストーリー**: 生産実績でCラインの問題を把握した田中さんが、設備の状態を詳しく確認する。

**使用するスマートカード**: `直近24時間の設備アラート`

**操作手順**:
1. 「直近24時間の設備アラート」をクリック

**期待される回答のポイント**:
- WLD-003からWARNING/CRITICALアラートが多発
- 振動異常、温度上昇、モーター負荷などのアラート
- 同一設備での繰り返しパターンを検出
- 推奨アクション：即時点検、生産停止検討

**確認できるデータ**:
- BigQuery: `equipment_alerts` テーブル

---

### シーン3: 非構造化データからの予兆検知（予知保全視点）

**ストーリー**: 田中さんが点検報告書を確認し、問題の経緯を把握する。

**使用するスマートカード**: `直近7日間の点検報告サマリ`

**操作手順**:
1. 「直近7日間の点検報告サマリ」をクリック

**期待される回答のポイント**:
- **WLD-003の状態が日々悪化していることを時系列で検出**
- 振動値：2.3→2.8→3.2→3.5mm/s と上昇トレンド
- 点検者のコメント「要経過観察」「次回詳細測定推奨」「緊急対応必要」
- 3/12時点で予兆があったことを指摘

**確認できるデータ**:
- GCS: `inspection_reports/` 配下の4ファイル
- 非構造化データ（テキスト）をAIが分析

**デモのハイライト**:
> 「AIが点検報告書のテキストを読み取り、数値の悪化トレンドと点検者の所見から予兆を検出しています」

---

### シーン4: 品質への影響確認（品質管理視点）

**ストーリー**: 設備劣化が品質にどう影響しているか確認する。

## 7. デモのポイント

### 7-1. 強調すべき価値

1. **構造化 × 非構造化データの統合分析**
   - BigQueryの数値データとGCSのテキストデータを横断して分析

2. **予兆検知の実演**
   - 点検報告書から振動値の悪化トレンドを自動検出

3. **多角的な視点での分析**
   - 生産・品質・保全・安全の4領域から同一問題を分析

4. **意思決定支援**
   - 単なるレポートではなく、具体的なアクションを提案

### 7-2. 想定Q&A

**Q: このデータはリアルタイムですか？**
A: デモ環境のため静的データですが、本番ではBigQueryへのリアルタイム連携が可能です。

**Q: 点検報告書はどうやって取り込むのですか？**
A: Cloud Storageにアップロードするだけで、AIが自動的に読み取り分析します。

**Q: 他の工場にも展開できますか？**
A: はい、企業フォルダを追加するだけで複数工場に対応できます。
