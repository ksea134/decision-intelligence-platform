# DIP チャット対話デモシナリオ
## 工場長の朝 〜 問題発見から意思決定まで

---

## シナリオ概要

**設定**: 2026年3月19日（水）朝8:30、工場長の田中さんが出勤し、DIPを使って今日の状況を確認する。

**所要時間**: 約15分

**ゴール**: 対話を通じてWLD-003の問題を発見し、原因を特定し、対策を決定する

---

## 対話シナリオ（10ステップ）

### Step 1: 朝の挨拶（アイスブレイク）

**入力**:
```
おはよう。今日の工場の状況を簡単に教えて。
```

**期待される回答のポイント**:
- 全体の生産状況サマリ
- **Cラインに問題があることを示唆**
- BigQueryから直近データを取得して回答

**デモでの見せ方**:
> 「自然な挨拶から始めても、AIがデータを取得して状況を報告します」

---

### Step 2: 異常の発見

**入力**:
```
Cラインに何か問題があるの？詳しく教えて。
```

**期待される回答のポイント**:
- Cラインの生産達成率が約50%と大幅低下
- 原因として**溶接ロボットWLD-003の故障**を特定
- 3月10日頃から悪化傾向

**デモでの見せ方**:
> 「AIが自動的に原因となる設備を特定しています」

---

### Step 3: 設備状態の確認

**入力**:
```
WLD-003の状態を詳しく分析して。
```

**期待される回答のポイント**:
- 設備アラート（WARNING/CRITICAL）の多発
- エネルギー消費が通常の+15〜20%
- 点検報告書からの所見引用
- **構造化データと非構造化データの両方を使用**

**デモでの見せ方**:
> 「BigQueryのアラートデータと、GCSの点検報告書テキストを組み合わせて分析しています」

---

### Step 4: 時系列での変化確認

**入力**:
```
WLD-003はいつから調子が悪くなった？経緯を時系列で教えて。
```

**期待される回答のポイント**:
- 3/12: 振動2.3mm/s、軽微な異音（要経過観察）
- 3/14: 振動2.8mm/s、異音増加
- 3/16: 振動3.2mm/s、**基準値超過**
- 3/18: 振動3.5mm/s、緊急対応必要
- **点検報告書から数値を抽出して時系列化**

**デモでの見せ方**:
> 「AIが複数の点検報告書を読み取り、数値の変化を時系列で整理しています」

---

### Step 5: 予兆の確認

**入力**:
```
3月10日より前に予兆はなかった？見逃したサインはある？
```

**期待される回答のポイント**:
- 3/12の点検で「軽微な異音」「要経過観察」の記載あり
- この時点で詳細点検を行っていれば早期発見できた可能性
- **予知保全の観点からの改善提案**

**デモでの見せ方**:
> 「過去データを遡って予兆を分析し、改善点を提案しています」

---

### Step 6: 品質への影響確認

**入力**:
```
この設備故障は品質にどう影響している？
```

**期待される回答のポイント**:
- Cラインの溶接不良が3/10以降増加
- 不良率の具体的な数値
- **顧客クレームとの関連**（TOYOTAから溶接関連クレーム）
- 品質コスト（手直し、廃棄）の試算

**デモでの見せ方**:
> 「設備→品質への因果関係をデータで示しています」

---

### Step 7: 安全への影響確認

**入力**:
```
安全面でのリスクは？ヒヤリハットとか起きてない？
```

**期待される回答のポイント**:
- 3/12: 火花飛散
- 3/14: ロボット異常動作
- 3/16: 異音発生で作業者が報告
- **ハインリッヒの法則に基づくリスク評価**
- 重大事故の可能性を警告

**デモでの見せ方**:
> 「ヒヤリハット報告書を分析し、重大リスクを警告しています」

---

### Step 8: 顧客影響の確認

**入力**:
```
顧客への影響は？クレームは来てる？
```

**期待される回答のポイント**:
- TOYOTAから溶接関連クレームが1月、2月、3月と継続
- **同一原因の再発パターン**を検出
- 顧客との信頼関係へのリスク
- 納期遅延リスクのあるオーダー

**デモでの見せ方**:
> 「クレーム報告書から再発パターンを検出し、顧客リスクを評価しています」

---

### Step 9: 原因の総括

**入力**:
```
ここまでの情報を整理して。結局、根本原因は何？
```

**期待される回答のポイント**:
- **根本原因: WLD-003の経年劣化（振動系統の異常）**
- 影響範囲: 生産性↓、品質↓、安全リスク↑、顧客満足度↓
- 3/12時点の「要経過観察」判断が分岐点だった
- 予知保全体制の不備が拡大を招いた

**デモでの見せ方**:
> 「複数領域のデータを統合して根本原因を特定しています」

---

### Step 10: 対策の意思決定

**入力**:
```
で、今日私は何をすべき？優先順位をつけて具体的なアクションを教えて。
```

**期待される回答のポイント**:

**即時対応（本日中）**:
1. WLD-003の即時生産停止
2. Cライン製品のA/B/Dラインへの振り分け検討
3. TOYOTAへの状況説明と納期調整の連絡

**短期対応（今週中）**:
4. WLD-003の詳細診断と修理/交換判断
5. 該当ロットの全数検査実施
6. 安全柵の修繕完了確認

**中期対応（今月中）**:
7. 予知保全体制の見直し（振動監視の閾値設定）
8. 点検基準の改定（「要経過観察」の対応フロー明確化）
9. 設備更新投資の検討（経営会議への起案）

**デモでの見せ方**:
> 「データ分析だけでなく、具体的なアクションプランまで提案しています」

## デモ実施のコツ

### 1. 自然な対話を心がける
- スマートカードを使わず、あえて自由質問で進める
- 「教えて」「分析して」「どうすべき？」など日常的な言葉で

### 2. 段階的に深掘りする
- 最初から「WLD-003」と言わない
- AIが問題を発見→ユーザーが深掘り、という流れ

### 3. 複数データソースの活用を見せる
- 「BigQueryとGCSの両方を使っている」ことを適宜説明
- 特に非構造化データ（点検報告書、ヒヤリハット）の分析を強調

### 4. 意思決定支援を強調
- 単なるレポートではなく「何をすべきか」まで提案
- 優先順位付けと具体的なアクションが出ることを見せる
